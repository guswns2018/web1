1. 웹 애플리케이션 만들기 - 오리엔테이션

웹은 가장 중요한 기술중에 하나이다.

여러 가지 기술들이 협력해서 웹이라고 하는 시스템을 동작하게 한다. (장점이자 단점)

낮은 진입장벽 (시작이 쉬울 뿐 만만한 분야는 아님)

서버(미들웨어 PHP, 데이터베이스 MySQL, 웹서버)와 클라이언트(HTML, CSS, JavaScript, 웹브라우저)가 어떻게 상호작용하는지 알아보는 것이 수업의 목적

나무를 통해서 숲을 보는 것이 목적!

무엇을 모르는지 모르는 상태에서 무엇을 모르는지 아는 상태로 넘어가는 것이 목적이다.

약 20~30시간 소요 예정

모든 부품을 다 다루진 않고, 부품을 어떻게 결합하는지 결합 방법을 배우는 것이다.

조립을 위해 필요한 부품들은 나중에 배우는 것이고, 결합 방법을 배우는 것이 더 중요.

3. 웹 애플리케이션 만들기 - 웹 애플리케이션을 만드는 순서

구상 : 무엇을 만들 것인가?

기획(계획) : 좀 더 세부적인 실행계획을 세워 가는 단계

무엇을 만들 것인지 좀 더 분명하게 계획하고, 어떤 기능들이 있어야 하는지 등도 포함됨

디자인 : 웹사이트를 좀 더 아름답게 하는 것 또는 사용자가 편리하게 이용할 수 있도록 함

개발_Development : 실제 서비스가 동작하도록 하는 것.

테스트 : 최종의 단계. 큰 조직의 경우에는 QA라고 하는 별도의 직군이 있음.

최종적으로 완벽히 될 때까지 이 과정을 반복함.

즉, Start Small 정신을 가져야 함.

복잡도가 작은게 중요함. (ex: 체크박스가 1개라면 2번의 테스트를 하면 되지만, 50개로 늘어나면 거의 천조 번의 테스트가 필요하다.)

따라서, 기능 하나가 추가되는 것은 지수폭발을 초래하기 때문에, 기능을 추가할 때 마다 신중히 고려해야 한다.
